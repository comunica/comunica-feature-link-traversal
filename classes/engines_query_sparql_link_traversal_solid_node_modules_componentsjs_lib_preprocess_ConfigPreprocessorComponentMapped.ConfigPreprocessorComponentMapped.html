Class ConfigPreprocessorComponentMapped

Handles config that refer to a component as type. The component may have parameters that can be applied on the config. Additionally, the component applies a custom constructor arguments mapping for its parameters.

    Map config with param instantiations to a raw config according to the given constructor arguments definition.

    For example, the constructor arg { key: '"param0"', value: 'http://example.org/param0' } with config { 'http://example.org/param0': '"abc"' } will be mapped to { key: '"param0"', value: '"abc"' }.

inheritParameterValues

  • inheritParameterValues(config: Resource, component: Resource): void

  • Inherited from ConfigPreprocessorComponent.inheritParameterValues

    • Defined in engines/query-sparql-link-traversal-solid/node_modules/componentsjs/lib/preprocess/ConfigPreprocessorComponent.d.ts:39

    Let this config inherit parameter values from previously instantiated configs. This will check for inheritanceValues that are defined on the component, which can refer to parameters from other components.

    For example, assume we had previously instantiated a component X with param P set to 'value'. If we instantiate component Y, which is defined to inherit values from param P of X, then it will automatically inherit this param P set to 'value'.

    This can effectively mutate the given config resource.

    Parameters

    • config: Resource

      The config

    • component: Resource

      The component

    Returns void
