expandTerm

  • expandTerm(term: string, expandVocab?: boolean, options?: IExpandOptions): null | string
    • Defined in node_modules/jsonld-context-parser/lib/JsonLdContextNormalized.d.ts:34

    Expand the term or prefix of the given term if it has one, otherwise return the term as-is.

    This will try to expand the IRI as much as possible.

    Iff in vocab-mode, then other references to other terms in the context can be used, such as to myTerm:

    {
  "myTerm": "http://example.org/myLongTerm"
}
    throws

    If the term is aliased to an invalid value (not a string, IRI or keyword).

    Parameters

    • term: string

      A term that is an URL or a prefixed URL.

    • Optional expandVocab: boolean

      If the term is a predicate or type and should be expanded based on @vocab, otherwise it is considered a regular term that is expanded based on @base.

    • Optional options: IExpandOptions

      Options that define the way how expansion must be done.

    Returns null | string

    The expanded term, the term as-is, or null if it was explicitly disabled in the context.
