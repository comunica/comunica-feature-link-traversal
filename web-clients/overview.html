Default

Web client

Default configuration with all available Link Traversal features.

Default Solid

Web client

Default configuration with Link Traversal features specific to traversing over one or multiple Solid data pods.

Default Solid (Adaptive)

Web client

Default configuration with Link Traversal features specific to traversing over one or multiple Solid data pods, using adaptive query optimization.

Solid Single pod

Web client

Configuration with Link Traversal features specific to traversing within a single Solid data pods.

Solid Source Provenance

Web client

Configuration with Link Traversal features enabling GRAPH-based origin document annotation when traversing over Solid data pods.

Solid Shapetrees

Web client

Configuration with Link Traversal features specific to Solid (data pods) with Shape Trees.

Follow All

Web client

Configuration that implements the cAll reachability semantics, and follows all possible links.

Follow Match Query

Web client

Configuration that implements the cMatch reachability semantics, and follows links in triples that match at least triple pattern in the query.

Follow Match Pattern Bound

Web client

Configuration that implements the cMatchPatternBound reachability semantics, and follows links in triples that match the current triple pattern leaf in the adaptive query plan.

Follow Content Policies (Restrictive)

Web client

Configuration that extracts SCL policies, and only follows links matching those policies.

Follow Content Policies (Conditional)

Web client

Configuration that extracts SCL policies, follows links according to the cMatch reachability semantics, but only allows links to be followed if they also match at least one SCL policy.

TREE

Web client

Configuration that follow all the relations in a TREE document.
